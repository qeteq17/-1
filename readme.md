# TeamFinder — курсовой проект, вариант 2

Веб-приложение для поиска участников в команду. Пользователь может зарегистрироваться, создать проект, присоединиться к чужому проекту, просматривать участников платформы и находить людей по навыкам.

Тема проекта: **«Разработка веб-приложения TeamFinder для поиска участников в команду»**.

Автор: **Алексеев Антон Вячеславович**.  
Преподаватель: **Мангушева Ляйля Сайяровна**.

## Что реализовано

- регистрация, вход, выход и смена пароля пользователей;
- редактирование профиля с аватаром, описанием, телефоном и ссылкой на GitHub;
- автоматическая генерация начальной аватарки по первой букве имени;
- создание, редактирование и завершение проектов;
- участие пользователей в проектах;
- главная страница со списком проектов от новых к старым;
- страница участников платформы;
- страница проекта с автором, описанием, ссылкой на GitHub, статусом и участниками;
- страница пользователя с контактами, проектами и навыками.

## Новая реализация по варианту 2

В проект добавлен модуль пользовательских навыков. Он раскрывает тему поиска участников в команду, потому что позволяет не просто просматривать список людей, а находить участников с нужной компетенцией.

Реализовано:

- модель `Skill` с обязательным полем `name` длиной до 124 символов;
- связь `User.skills` с моделью `Skill`;
- обратная связь `skill.users` для получения пользователей, владеющих выбранным навыком;
- отображение навыков в профиле пользователя;
- добавление навыков владельцем профиля без перезагрузки страницы;
- удаление навыков владельцем профиля без удаления самого навыка из базы данных;
- автодополнение навыков через эндпоинт `GET /users/skills/?q=<начало навыка>`;
- создание нового навыка через `POST /users/<id>/skills/add/`;
- добавление существующего навыка через `skill_id`;
- удаление навыка через `POST /users/<id>/skills/<skill_id>/remove/`;
- фильтрация списка участников по навыку через `/users/list/?skill=<название навыка>`;
- подсветка активного фильтра и сброс фильтра.

## Соответствие заданию

Основные модели:

- `Project.name`, `description`, `owner`, `created_at`, `github_url`, `status`, `participants`;
- `User.email`, `name`, `surname`, `avatar`, `phone`, `github_url`, `about`, `is_active`, `is_staff`, `skills`;
- `Skill.name`.

Ключевые обратные связи:

- `user.owned_projects` — проекты, которыми владеет пользователь;
- `user.participated_projects` — проекты, в которых пользователь участвует;
- `skill.users` — пользователи, владеющие навыком.

Валидация:

- телефон сохраняется в формате `+7XXXXXXXXXX`;
- принимаются только исходные форматы `8XXXXXXXXXX` и `+7XXXXXXXXXX`;
- номера проверяются на уникальность с учетом вариантов `8...` и `+7...`;
- ссылка на GitHub должна быть валидной ссылкой и вести на домен `github.com`.

## Установка и запуск

1. Установить зависимости:

```bash
pip install -r requirements.txt
```

2. Запустить PostgreSQL. Можно использовать Docker:

```bash
docker compose up -d
```

3. Проверить файл `.env`:

```env
POSTGRES_DB=teamfinder
POSTGRES_USER=postgres
POSTGRES_PASSWORD=123321A
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
DJANGO_SECRET_KEY=dev-secret-key
DJANGO_DEBUG=True
ALLOWED_HOSTS=127.0.0.1,localhost
```

4. Применить миграции:

```bash
python manage.py migrate
```

5. При необходимости создать демонстрационные данные:

```bash
python manage.py seed_demo
```

6. Запустить приложение:

```bash
python manage.py runserver
```

Адрес приложения: `http://127.0.0.1:8000/`.

## Основные URL

- `/` — редирект на список проектов;
- `/project/list/` — совместимый редирект на список проектов по формулировке задания;
- `/projects/list/` — список проектов;
- `/projects/create-project/` — создание проекта;
- `/projects/<id>/` — страница проекта;
- `/projects/<id>/edit/` — редактирование проекта;
- `/users/register/` — регистрация;
- `/users/login/` — вход;
- `/users/logout/` — выход;
- `/users/list/` — список участников;
- `/users/list/?skill=Python` — фильтрация участников по навыку;
- `/users/<id>/` — профиль пользователя;
- `/users/skills/?q=Py` — автодополнение навыков.
